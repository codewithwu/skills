# 项目开发规范

## 环境配置

1. **激活虚拟环境**：项目开始阶段需要先激活虚拟环境，使用以下命令：
   ```bash
   source .venv/bin/activate
   ```

2. **包管理**：必须使用 `uv` 在虚拟环境中安装和管理所有依赖包，禁止使用 pip 直接安装包。
   ```bash
   uv add <package-name>
   uv sync
   ```

## 代码开发规范

1. **代码风格**：
   - Python 使用 PEP 8 风格指南
   - 使用 4 空格缩进（不要使用 Tab）
   - 每行最大长度不超过 120 字符
   - 文件末尾保留一个空行

2. **命名规范**：
   - 变量和函数：`snake_case`（如 `get_user_name`）
   - 类名：`PascalCase`（如 `UserService`）
   - 常量：`UPPER_SNAKE_CASE`（如 `MAX_RETRY_COUNT`）
   - 私有属性：以 `_` 开头（如 `_internal_cache`）

3. **函数设计**：
   - 函数应保持单一职责，每个函数只做一件事
   - 函数长度尽量控制在 50 行以内
   - 优先使用明确的参数名调用（关键字参数）
   - 始终提供函数的类型注解（Type Hints）

4. **Import 规范**：
   - 标准库、第三方库、本地模块之间用空行分隔
   - 按 `标准库 → 第三方库 → 本地模块` 的顺序排列
   - 使用绝对导入，避免相对导入的歧义

5. **注释与文档**：
   - 使用 docstring 描述模块、类和公共函数的作用
   - 复杂逻辑添加行内注释说明
   - 避免显而易见的注释，保持注释的价值

6. **Git 提交规范**：
   - 提交信息使用中文，简洁描述修改内容
   - 格式：`类型: 描述`（如 `fix: 修复用户登录失败问题`）
   - 类型可选：`feat`/`fix`/`docs`/`style`/`refactor`/`test`/`chore`

# 短视频脚本创作技能

## 概述

本技能用于创作 **30秒短视频脚本**，分 **3个连贯场景**，每场景 **10秒**，角色以 **动物** 为主。

适用于豆包AI等文字转视频工具，需要提供清晰、具体的画面描述。

遵循"**快节奏、高密度、强钩子**"原则——用户注意力极其有限，前3秒定生死。

---

## 核心规则

### 黄金前3秒：暴力抓人
- 不要铺垫，直接抛出冲突、悬念或视觉刺激
- 避免：开场介绍、背景交代、寒暄

### 结构：起-转-合（极简版）
| 时间 | 阶段 | 核心任务 |
|------|------|----------|
| 0-10秒 | 抛出钩子 | 痛点、好奇、反差（只留"高能"） |
| 10-20秒 | 核心内容 | 只讲一个信息点，讲透 |
| 20-30秒 | 收尾引导 | 引导互动（点赞/评论/关注） |

### 内容做减法
- 删除所有过渡、寒暄、铺垫
- 30秒只把 **1个精彩信息** 讲透

### 爆款公式参考
1. **实用类**：问题 + 错误示范 + 正确方法 + 对比效果
2. **情绪类**：情绪铺垫 + 转折事件 + 强烈反应
3. **反差类**：普通开始 + 快速变化 + 惊艳结果

### AI创作适配要求
- 画面描述要具体、场景化，便于AI理解执行
- 每个场景采用少切换的单镜头描述
- 台词直接融入画面描述中，格式："角色动作+台词"
- 重视角色一致性，每个场景需重复描述角色特征

## 工作流程

### 第一阶段：头脑风暴（至少3轮）

**第1轮提问** - 了解基本信息：
1. 这个短视频想传达什么**核心信息**或**主题**？（如：坚持、友情、勇气、环保等）
2. 你想用哪种**动物主角**？（可指定具体动物如猫咪、狗狗、兔子等）
3. 目标**受众**是谁？（如：年轻人、宝妈、学生、上班族等）
4. 希望是什么**风格**？（如：搞笑、感人、治愈、反转、励志等）

**第2轮提问** - 深化细节：
1. 视频开头打算用什么**钩子**？（如：一句话、一个画面、一个冲突）
2. 有没有特别想要或想要**避免**的元素？
3. 希望传达什么**情感**让观众产生共鸣？
4. 视频的**结局/彩蛋**有什么想法吗？

**第3轮提问** - 确认与完善：
1. 这个视频的**亮点/记忆点**是什么？
2. 是否有特定的**BGM风格**或**画面风格**偏好？
3. 有没有参考的**爆款视频**或**同类型作品**？
4. 还有其他需要补充的**关键要素**吗？

> **提示**：如果用户在某个问题上已经给出明确答案，可以跳过该问题并标记"✅ 已确认"，直接进入下一轮。

### 第二阶段：生成脚本

在完成至少3轮头脑风暴后，基于收集到的信息生成最终的短视频脚本。

### 第三阶段：AI角色图片生成提示词

为了确保视频中角色的一致性，需要为所有出现的角色生成**三视图**（正面、侧面、背面），供AI绘图工具使用。

**操作步骤**：
1. 识别脚本中所有出现的角色（包括主角和配角）
2. 为每个角色撰写三视图提示词
3. 提示词应包含：角色外观描述、视角标识、背景要求、画风要求

**提示词模板**：
```
[动物类型] character design,三视图(正面、侧面、背面), white background, [画风],
[体型描述], [毛色描述], [服装配饰], full body, character sheet,
consistent proportions, clean lines, character turn-around sheet,
front view side view back view, anime style, high quality
```

**输出格式**：
```markdown
## [角色名称]角色提示词

```
[完整的三视图提示词]
```

## 输出格式

最终输出的脚本必须包含：

### 1. 基础信息表
```markdown
| 项目 | 内容 |
|------|------|
| 主题 | xxx |
| 动物主角 | xxx |
| 场景数 | 3个 |
| 单场景时长 | 10秒 |
| 目标受众 | xxx |
| 风格 | xxx |
| 结构类型 | 实用类/情绪类/反差类 |
```

### 2. 分镜脚本（段落形式）

```markdown
## 场景一（第1-10秒）：场景名称

主角：[角色外观描述，包括体型、毛色、眼睛、特色标记等]

[具体画面描述，台词直接融入，格式："角色动作说："台词""]

[具体画面描述，台词直接融入]

[具体画面描述，台词直接融入]

## 场景二（第11-20秒）：场景名称

主角：[角色外观描述]

[具体画面描述，台词直接融入]

[具体画面描述，台词直接融入]

[具体画面描述，台词直接融入]

## 场景三（第21-30秒）：场景名称

主角：[角色外观描述]

[具体画面描述，台词直接融入]

[具体画面描述，台词直接融入]

[具体画面描述，台词直接融入]
```

### 3. 角色一致性说明

```markdown
全篇统一特征：[列出所有角色外观特征]
表情系统：[不同情绪对应的表情描述]
动作习惯：[角色特有的动作习惯]
```

> **重要**：每个场景都需要包含主角描述和台词，确保AI生成时角色一致性

### 4. 创意说明
- 简要说明脚本的创意亮点
- 解释为什么这个开头能"暴力抓人"
- 说明如何引导观众互动

### 5. 执行建议（可选）
- 拍摄技巧
- 动物训练建议
- 后期剪辑要点

---

## 保存结果

将最终确认的脚本以 **markdown 格式** 保存到用户指定位置或当前目录，文件名格式：

```
短视频脚本_[主题]_[日期].md
```

例如：`短视频脚本_猫咪学游泳_20260414.md`

## 质量检查清单

生成脚本后自检：
- [ ] 前3秒是否有强钩子？
- [ ] 每个场景是否有明确目标？
- [ ] 是否只有单一信息点？
- [ ] 是否有引导互动的结尾？
- [ ] 台词是否简洁有力？
- [ ] 动物角色的动作/表情是否可执行？
- [ ] 场景之间是否连贯？
- [ ] 每个场景是否包含角色描述和台词？
- [ ] 角色特征在场景间是否保持一致？
- [ ] 是否已为所有角色生成三视图提示词？
